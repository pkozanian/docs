pywren.wait module¶

pywren.wait.wait(fs, return_when=1, THREADPOOL_SIZE=64, WAIT_DUR_SEC=5)¶

this will eventually provide an optimization for checking if a large number of futures have completed without too much network traffic by exploiting the callset

From python docs:

Wait for the Future instances (possibly created by different Executor instances) given by fs to complete. Returns a named 2-tuple of sets. The first set, named “done”, contains the futures that completed (finished or were cancelled) before the wait completed. The second set, named “not_done”, contains uncompleted futures.

http://pythonhosted.org/futures/#concurrent.futures.wait
